06. Soalan & Jawapan (Q&A)

Fahami logik pengendalian sistem dan struktur hos bagi pemantapan teori pembangunan.

Q: Mengapa kita memasang Docker terus di dalam Ubuntu WSL berbanding menggunakan aplikasi Windows Docker Desktop?

A: Memasang terus di dalam WSL jauh lebih ringan kerana ia tidak memuatkan aplikasi antaramuka (GUI) Windows yang berat. Ini menjimatkan penggunaan RAM sistem laptop secara drastik serta memberikan kelajuan prestasi penulisan disk I/O yang hampir sama seperti pelayan Linux sebenar.

Q: Adakah kontena akan terpadam secara automatik jika saya menutup terminal Ubuntu atau mematikan laptop?

A: Tidak. Kontena tersebut hanya akan bertukar status kepada ditutup (Stopped). Apabila anda membuka semula terminal dan menghidupkan servis Docker daemon, anda boleh menjalankan arahan docker start nama-container untuk menyambung semula kerja tanpa kehilangan data.

Q: Bagaimana sistem Windows boleh mengakses `localhost:8080` sedangkan kontena dijalankan di dalam subsystem Linux?

A: Ini dibantu oleh ciri auto-mapping (Port Forwarding) binaan dalam WSL2. Sistem WSL2 secara automatik memetakan sebarang port rangkaian pelayan yang diaktifkan di dalam Linux terus ke antara muka rangkaian localhost Windows hos anda.
